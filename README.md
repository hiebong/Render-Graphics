## Features

### Client Side

- **Responsive Design** — Fully responsive on mobile, tablet, and desktop
- **Portfolio Gallery** — Browse artworks with category filters
- **Services Page** — View available services with pricing
- **Order Form** — Place custom commissions with reference images (up to 10 images)
- **Order Tracking** — Track order status using your Order ID
- **Email Notifications** — Receive confirmation and status update emails automatically

### Admin Dashboard

- **Secure Login** — Firebase Authentication protected
- **Order Management** — View, update status, and manage all incoming orders
- **Portfolio Management** — Upload, rename, and delete artworks
- **Services Management** — Add, edit, and delete offered services
- **Additionals Management** — Manage add-on services and options
- **Status Updates** — Send automated email notifications to customers on status changes

### Backend API (Node.js/Express)

- **Order Processing** — Save orders to Firestore and send confirmation emails
- **Status Updates** — Trigger status-change notifications to clients
- **Email Service** — Nodemailer with Gmail SMTP
- **RESTful Endpoints** — Clean API design with proper HTTP responses

## Installation & Setup

### Prerequisites

- Node.js (v18 or higher)
- A Firebase account
- A Gmail account (for email notifications)

### Step 1: Clone the Repository

```bash
git clone https://github.com/hiebong/Render-Graphics.git
cd Render-Graphics
```

### Step 2: Install Dependencies

```bash
npm install
```

### Step 3: Set Up Environment Variables

Create a `.env` file in the root directory:

```env
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-gmail-app-password
```

### Step 4: Get a Gmail App Password

1. Enable 2-Factor Authentication on your Google account
2. Go to: [https://myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
3. Select **"Mail"** as the app and **"Windows Computer"** as the device
4. Copy the generated 16-character password and paste it into `EMAIL_PASS`

### Step 5: Configure Firebase

1. Create a Firebase project at [https://console.firebase.google.com](https://console.firebase.google.com)
2. Enable **Authentication** (Email/Password provider)
3. Create a **Firestore** database
4. Copy your Firebase config object into all relevant HTML files

### Step 6: Run Locally

```bash
# Start the backend server
npm run dev

# Open the frontend with Live Server
# Visit: http://localhost:5500
```

## API Endpoints

| **Method** | **Endpoint**        | **Description**            | **Request Body**                                      |
| ---------- | ------------------- | -------------------------- | ----------------------------------------------------- |
| `POST`     | `/api/order`        | Submit a new order         | `{ name, email, order_id, services, message, total }` |
| `POST`     | `/api/order-status` | Send a status update email | `{ orderId, name, email, status }`                    |
| `GET`      | `/api/health`       | Health check               | None                                                  |

### Example Request

```bash
curl -X POST https://render-graphics.vercel.app/api/order \
  -H "Content-Type: application/json" \
  -d '{"name":"Juan dela Cruz","email":"juan@example.com","order_id":"ABC123","services":"Pet Portrait","total":"₱500"}'
```

### Example Response

```json
{
  "success": true,
  "message": "Order received and confirmation email sent.",
  "order_id": "ABC123"
}
```

## Firebase Security Rules

Add these rules to your Firestore to restrict write access to authenticated admins only:

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Orders — anyone can create, only admins can read/update/delete
    match /orders/{orderId} {
      allow create: if true;
      allow read, update, delete: if request.auth != null;
    }

    // Portfolio — only authenticated admins
    match /portfolio/{item} {
      allow read: if true;
      allow write: if request.auth != null;
    }

    // Services — only authenticated admins
    match /services/{service} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
```

## Deployment

This project is deployed on **Vercel** (frontend + backend as serverless functions).

### Deploy to Vercel

1. Push your project to GitHub
2. Go to [https://vercel.com](https://vercel.com) and import your repository
3. Add the following **Environment Variables** in the Vercel dashboard:

| Variable     | Value                     |
| ------------ | ------------------------- |
| `EMAIL_USER` | `your-email@gmail.com`    |
| `EMAIL_PASS` | `your-gmail-app-password` |

4. Vercel auto-deploys on every push to `main`

### `vercel.json` Configuration

```json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/index.js" },
    { "source": "/(.*)", "destination": "/$1" }
  ]
}
```

## How It Works

```
Client Places Order
       ↓
Order saved to Firestore
       ↓
Confirmation email sent via Nodemailer
       ↓
Admin reviews in Dashboard
       ↓
Admin updates status (Pending → On Progress → Completed)
       ↓
Client receives status update email
       ↓
Art delivered digitally
```
